Detailed Description

Texture object for OpenGL texturing.

The SLGLTexture class implements an OpenGL texture object that can is used by the SLMaterial class. A texture can have 1-n SLImages in the vector _images. A simple 2D texture has just a single texture image (_images[0]). For cube maps you will need 6 images (_images[0-5]). For 3D textures you can have as much images of the same size than your GPU and/or CPU memory can hold. The images are not released after the OpenGL texture creation. They may be needed for ray tracing.

void SLGLTexture::bindActive ( SLint  texID = 0)

SLGLTexture::bindActive binds the active texture. This method must be called by the object that uses the texture every time BEFORE the its rendering. The texID is only used for multi texturing. Before the first time the texture is passed to OpenGL.

void SLGLTexture::build ( SLint  texID = 0)

Builds an OpenGL texture object with the according OpenGL commands. This texture creation must be done only once when a valid OpenGL rendering context is present. This function is called the first time within the enable method which is called by object that uses the texture.
